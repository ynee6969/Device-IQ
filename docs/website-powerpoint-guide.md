# DeviceIQ PowerPoint Creation Guide

## Recommended Slide Count

Use 6 to 8 slides. That is enough to explain the website clearly without making the presentation feel crowded.

## Recommended Flow

1. Title
2. Problem
3. Website Solution
4. Key Pages
5. UI/UX and Responsiveness
6. Closing

## Suggested Slide Titles

- DeviceIQ
- Why We Built It
- The User Problem
- Main Website Flow
- UI/UX Design Choices
- Responsive Layout
- Conclusion

## What To Place In Each Slide

### Slide 1: Title
- Project name
- Short subtitle
- Your names
- Section and date
- One strong homepage screenshot

### Slide 2: Why We Built It
- 2 or 3 short bullets about phone-shopping confusion
- One clean icon or diagram
- Keep text brief

### Slide 3: The User Problem
- Too many phone choices
- Specs scattered across sites
- Hard to compare quickly
- Use a simple visual flow

### Slide 4: Main Website Flow
- Homepage
- Dashboard
- Compare page
- Favorites
- Phone detail page

### Slide 5: UI/UX Design Choices
- Dark glassmorphism style
- Green accent color
- Card-based layout
- Clear typography hierarchy
- Optional sign-in

### Slide 6: Responsive Design
- Desktop header
- Mobile bottom tab bar
- Filter panel behavior
- Compare page stacking on smaller screens

### Slide 7: Conclusion
- One sentence summary
- One final homepage or compare screenshot
- Thank you message

## Best Screenshots To Use

- Homepage hero
- Dashboard with phone cards
- Compare page hero
- Phone detail page
- Mobile header or bottom tab bar
- Theme settings modal if you want to show polish

## Best Sections To Showcase

- Hero section for the first impression
- Dashboard for browsing
- Compare page for the signature feature
- Phone detail page for spec depth
- Favorites for account-based saving

## How To Explain Responsiveness Visually

- Show desktop and mobile screenshots side by side.
- Point out the sticky header on desktop.
- Point out the mobile bottom nav and action buttons.
- Mention that grids collapse into one column when screen space is smaller.

## Recommended Animations And Transitions

- Use simple fade or zoom for slide changes.
- Avoid flashy transitions.
- Keep motion subtle so the content stays readable.

## Layout Recommendations

- Use one main visual per slide.
- Keep bullet lists short.
- Leave a lot of white space.
- Do not overload slides with paragraphs.

## Typography Recommendations

- Use one font family for headings and one for body text.
- Keep headings large and bold.
- Keep body copy short and readable.
- Match the modern, clean tone of the website.

## Color Palette Suggestions

- Dark charcoal or navy background
- Green accent for highlights
- Light text on dark surfaces
- Small blue accent if needed for secondary emphasis

## Design Consistency Tips

- Match the slide colors to the site colors.
- Reuse the same icon style across slides.
- Keep spacing consistent.
- Use the same type scale across the deck.

## Minimalist Presentation Strategy

- One idea per slide.
- No long paragraphs.
- Use screenshots instead of explanations where possible.
- Let the website visuals do most of the talking.

## Technical Presentation Best Practices

- Explain what the user experiences, not every line of code.
- Talk about the rationale behind the layout.
- Use the same terms from the website: browse, compare, favorites, responsiveness.
- If questioned, answer with the user benefit first and the technical detail second.

## Presenter Tips

- Do not rush the opening.
- Point to the screenshot before speaking about it.
- Keep your voice steady.
- If you lose your place, move to the next slide instead of stopping.

## Eye Contact Tips

- Look at the panel for the first sentence.
- Look at the screen only when pointing to a feature.
- Return to eye contact after each short explanation.

## Voice Projection Tips

- Speak slightly slower than normal conversation.
- Project the end of your sentence.
- Pause after every major idea.

## How To Answer Difficult Instructor Questions

- Start with the reason behind the design.
- If asked about responsiveness, mention the layout changes on mobile.
- If asked about UI/UX, mention clarity, contrast, and faster decision-making.
- If asked about a feature, connect it to the user problem it solves.

## Common PowerPoint Mistakes

- Too much text on one slide
- Small screenshots that are hard to see
- Weak contrast between text and background
- Too many animations
- Slides that repeat the same idea

## What Not To Put On Slides

- Full code blocks
- Long paragraphs
- Too many bullet points
- Unnecessary backend details
- Irrelevant stock photos

## Recommended Visual Hierarchy

- Big title first
- Screenshot second
- Short bullets last

## Recommended Text Limits Per Slide

- Title: 5 to 8 words
- Subtitle: 1 short line
- Bullets: 3 to 5 bullets
- Each bullet: 1 short sentence

## Best Modern Presentation Style

- Clean editorial layout
- Dark theme with accent highlights
- Large screenshots
- Simple icon accents

## Suggested Canva Or PowerPoint Theme

- Dark tech theme
- Minimal modern theme
- Editorial portfolio theme
- Clean product showcase theme

## Suggested Opening Slide Design

- Full-width homepage screenshot
- Large title in one corner
- Small subtitle under the title
- Dark background with green accent line

## Suggested Ending Slide Design

- Clean closing slide
- One short thank-you line
- Small logo or website screenshot
- No extra text
